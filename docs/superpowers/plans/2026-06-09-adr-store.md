# Replace ARCHITECTURE.md with a Lazy ADR Store — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the generated `docs/scifi/ARCHITECTURE.md` everywhere and replace it with a lazily-created `docs/scifi/adr/` decision-record store that spec/plan skills write to sparingly and grep on demand.

**Architecture:** A pure deletion + reframing change. The scaffold stops emitting `ARCHITECTURE.md`; the CLI stops listing it; skills stop reading it and instead grep `docs/scifi/adr/` (created lazily on first record). ADR authoring lives only in `sf-feature` and `sf-plan`. Review skills judge against `spec.md`/`design.md` and treat `CONTEXT.md` strictly as ubiquitous-language naming. No new CLI command, no migration.

**Tech Stack:** TypeScript (Node), Vitest, Biome. Skills are `body.md` + `manifest.ts` + template/dispatch `.md` files under `skills/<id>/`.

Spec: `docs/superpowers/specs/2026-06-09-adr-store-design.md`.

Decisions live in `docs/scifi/adr/` as numbered records `NNNN-slug.md`. The

- **Numbering.** Run `ls docs/scifi/adr/` and take `max + 1`, zero-padded (e.g.
